Sorting Algorithms Explained

A step-by-step guide to Bubble Sort, Selection Sort and Insertion Sort — with full worked examples, comparison counts and complexity analysis for KCSE and university level.

✍️ By Mwalimu Ronny📚 Secondary & University Level⏱️ 14 min read

What Is a Sorting Algorithm?

A sorting algorithm is a set of instructions that rearranges a list of items into a specific order — typically ascending (smallest to largest) or descending (largest to smallest). Sorting is one of the most fundamental operations in computing, and being able to sort data efficiently is crucial for tasks ranging from displaying contacts in alphabetical order to organising millions of search results in milliseconds.

For KCSE Computer Studies, you need to understand three sorting algorithms in detail: Bubble Sort, Selection Sort and Insertion Sort. Each works differently, has different performance characteristics, and suits different situations.

Key terminology: A comparison is when two elements are checked to see which is larger. A swap is when two elements exchange positions. The number of comparisons and swaps determines how efficient a sorting algorithm is.

Bubble Sort

Bubble Sort is the simplest sorting algorithm to understand. It works by repeatedly stepping through the list, comparing adjacent pairs of elements and swapping them if they are in the wrong order. After each complete pass through the list, the largest unsorted element "bubbles up" to its correct position at the end.

The algorithm repeats this process, each time ignoring the last element (which is now sorted), until no swaps are needed in a complete pass — at which point the list is fully sorted.

Time complexity: O(n²) in the worst and average case. For 5 elements, the maximum number of comparisons is 4+3+2+1 = 10. For n elements it is n(n-1)/2 comparisons.

Selection Sort

Selection Sort works by dividing the list into a sorted portion (initially empty, on the left) and an unsorted portion (initially the whole list). In each pass, it selects the smallest element from the unsorted portion and places it at the end of the sorted portion. Unlike Bubble Sort, Selection Sort makes at most one swap per pass.

Key advantage of Selection Sort over Bubble Sort: fewer swaps. This is important when swapping items is expensive (e.g. swapping large records in a database).

Insertion Sort

Insertion Sort works the way most people sort playing cards in their hand. It takes one element at a time from the unsorted portion and inserts it into its correct position in the already-sorted portion, shifting other elements along to make room.

Insertion Sort is particularly efficient when the list is already nearly sorted — it can sort such a list in close to O(n) time.

Comparing the Three Algorithms

AlgorithmBest CaseAverage CaseWorst CaseSwaps
Bubble SortO(n)O(n²)O(n²)Up to O(n²)
Selection SortO(n²)O(n²)O(n²)O(n) — at most 1 per pass
Insertion SortO(n)O(n²)O(n²)Up to O(n²)

All three algorithms have O(n²) worst-case performance, which makes them inefficient for large datasets. More advanced algorithms like Merge Sort and Quick Sort achieve O(n log n), but those are university-level topics.

For the KCSE syllabus, the key points are: Bubble Sort is simplest but makes many swaps; Selection Sort makes fewer swaps; Insertion Sort is best when data is nearly sorted.

KCSE Examination Tips

  • When tracing a sort, always show each pass clearly with the array state after each swap or comparison.
  • Count comparisons and swaps separately — examiners often ask for one or both.
  • Know which algorithm is most efficient for nearly-sorted data (Insertion Sort) and which makes the fewest swaps (Selection Sort).
  • Bubble Sort's optimised version stops early if no swaps occur in a pass — mention this if asked about efficiency improvements.
